# Digital Wallet UX Improvement Project
**Role:** Lead Designer

## Overview
This project focuses on improving transaction usability in the finance domain by applying Human Computer Interaction (HCI) principles such as visibility, feedback, and error prevention. The finance domain was selected because digital wallet applications are widely used for daily transactions.

## Problem Identification
Many digital wallet applications do not clearly display the available balance during the money transfer process. When users enter an amount greater than their available balance, an error message appears, but the user does not know the exact balance and must navigate back to the home screen to check it. This causes confusion, interrupts the transaction flow, and increases user frustration.

## Proposed Solution
The proposed solution introduces a real-time balance display during the transaction process. There is an option on the transaction screen to see the current balance, which reduces user errors and unnecessary navigation.

## Research & Analysis
* **User Persona:** Designed for users like "Ayesha Khan," a 21-year-old university student who needs clear balance visibility and instant feedback during transactions.
* **Competitor Analysis:** Evaluated Easypaisa and JazzCash, identifying common weaknesses such as the balance not being visible during transactions and limited real-time feedback.

By displaying a real-time balance during transactions and providing instant feedback, this design reduces user errors, minimizes unnecessary navigation, and enhances the overall user experience.
